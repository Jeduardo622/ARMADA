# Gate Ownership & Thresholds

- G1 (Foundations): Owner PM; Criteria: sim skeleton runs; CI green; crash <1% in smoke; core ADRs accepted. QA sign-off required.
- G2 (Playable Loop): Owner PM; Criteria: missions 1–6 playable; progression loop; telemetry validated; perf within budget on target devices; flags/defaults verified. QA + Design sign-off.
- G3 (Pre-Soft Launch): Owner PM; Criteria: stability/perf thresholds met; store metadata ready; rollback paths tested; security/privacy checks passed. QA + Product + Security sign-off.
- Launch: Owner Product; Criteria: soft-launch region approved; store builds submitted; dashboards live; support/on-call rota set; major risks closed or accepted. PM coordinates sign-offs.

Sign-offs recorded in release notes; any exception must list owner and rationale.
