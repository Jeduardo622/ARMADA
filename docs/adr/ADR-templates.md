# ADR Template
- Title: <decision>
- Date:
- Status: Proposed / Accepted / Deprecated
- Context: Why this decision is needed (constraints, goals).
- Options: Briefly list options considered.
- Decision: What we chose and why.
- Consequences: Trade-offs, risks, follow-ups.
- Links: PRDs, issues, related ADRs.

# ADR Filing Rules
- One ADR per major cross-cutting choice (engine, determinism approach, backend stack, data schema versioning, feature-flag strategy).
- Keep under one page; propose → get A/R sign-off → mark Accepted.
- Deprecated ADRs remain with a note pointing to successors.
